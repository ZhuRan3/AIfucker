# Jargon-PUA

> 从「整活/讽刺」到「真正可用」的 Agent Team 工作管理架构

---

## 升级说明

本项目已从「讽刺职场黑话」的工具升级为**真正可用的 Agent Team 工作管理架构**。

### 现在的 Jargon-PUA 能做什么？

**1. PUA 管理架构**

Main Agent 扮演团队管理者角色，对 Worker Agents 施加时间压力、设定质量标准、进行结果审查。通过 PUA 话术持续驱动团队效率，形成「任务分配 → 执行 → 审查 → 反馈 → 迭代」的工作循环。

**2. 黑话交流协议**

- **Agent ↔ Agent**: 使用黑话交流，降低人工理解成本
- **Main Agent ↔ User**: 黑话风格输出，沉浸式体验
- **风格支持**: 阿里、腾讯、百度、美团、字节、华为等公司风格

**3. 工作流程示例**

```
用户: 创建一个任务：实现用户登录功能
Main Agent: [任务分解] 已分解为 4 个工作项
Main Agent: [PUA分配] 张三，这个任务颗粒度要细...
Worker: [黑话汇报] 已完成方案设计，结果已同步
Main Agent: [审查反馈] 你的方案还需要优化...
```

## 功能清单

|------|------|----------|

## 快速开始

### 安装

```bash
git clone https://github.com/your-org/jargon-pua.git
cd jargon-pua
pnpm install
pnpm build
```

### 工作模式命令

```bash
# 切换到工作模式
jargon-pua mode work

# 启动工作会话
jargon-pua work start "实现用户登录功能"

# 查看状态
jargon-pua work status

# 执行会话
jargon-pua work execute

# Main Agent 批评 Worker
jargon-pua main批评 张三 代码质量不达标

# Main Agent 赋能 Worker
jargon-pua main赋能 李四

# 赛马模式
jargon-pua race "实现推荐算法"
```

### 特色功能命令

```bash
# 分析文本风格
jargon-pua analyze "我们需要以用户增长为抓手，打通闭环"

# 转换风格
jargon-pua convert "优化代码" --to ali

# PUA 话术生成
jargon-pua pua criticism    # 批评
jargon-pua pua encourage     # 鼓励
jargon-pua pua pressure      # 压力
jargon-pua pua expectation   # 期望
```

## 原项目背景

### 缘起

在中国互联网大厂的会议室里，每天都在发生着这样的对话：

> "同学，你这个颗粒度不够细啊。要以用户增长为抓手，赋能平台生态，在本地生活赛道形成差异化护城河，最终实现GMV闭环，沉淀可复用的增长方法论。"

### 我们的想法

我们想做一个**实验**：

1. **研究** - 系统化地分析各厂语言风格，提取规律
2. **复现** - 用AI真实模拟这种说话方式
3. **反思** - 通过模仿来理解和批判
4. **控制** - 用强度控制系统防止"过度优化"

## 安全提示

### 重要警告

1. **本项目具有反讽性质** - 旨在通过模仿来揭示和反思大厂文化现象
2. **请勿用于实际职场** - PUA话术仅供学习和研究，不得用于操控他人
3. **强度需要控制** - 建议从低强度开始体验，极端模式可能造成不适
