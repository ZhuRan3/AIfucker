# Jargon-PUA Skill

> 同学，你这个颗粒度需要再细一点，找到合适的抓手，为项目赋能，形成价值闭环。

## 📖 技能介绍

**Jargon-PUA** 是一个集研究、分析、生成、应用于一体的综合平台。通过AI模拟大厂语言风格和职场PUA话术，我们希望：

1. **研究大厂文化** - 结构化分析各厂语言风格特征
2. **探索交互范式** - 用反讽的方式体验大厂沟通模式
3. **防止过度优化** - 通过强度控制系统保护代码质量
4. **创作行为艺术** - 每一次对话都是作品的一部分

## 🎯 核心功能

### 1. 模式切换

切换AI与你的交流方式：

```
@jargon-pua 开启黑话模式
@jargon-pua 开启PUA模式
@jargon-pua 开启完全模式（黑话+PUA）
@jargon-pua 关闭特殊模式
```

**效果示例**：

| 模式 | AI回复示例 |
|------|-----------|
| 正常 | "这个函数需要优化" |
| 黑话 | "这个函数的颗粒度不够细，要找到合适的抓手赋能业务" |
| PUA | "其实，我对你这个函数有一些失望的。当初写的时候，我是希望你能认真一点的。" |
| 完全 | "其实，我对你赋能这个函数的抓手有一些失望。颗粒度太粗，没有形成闭环，你要有owner意识。" |

### 2. 风格识别

识别文本属于哪个大厂风格：

```
@jargon-pua 识别这段话的风格：
"我们需要以用户增长为抓手，赋能平台生态，在本地生活赛道形成差异化护城河"
```

### 3. 风格转换

将文本转换为指定大厂风格：

```
@jargon-pua 把"我们讨论一下这个方案"转换成阿里风格
@jargon-pua 用腾讯风格重写这段话
@jargon-pua 把这段华为风格的话改成字节风格
```

### 4. PUA话术生成

生成各厂典型的PUA话术：

```
@jargon-pua 生成一段阿里风格的批评话术，强度中等
@jargon-pua 帮我写一段拒绝涨薪的话术，用美团风格
@jargon-pua 生成布置加班任务的话术，强度要轻一点
```

### 5. 黑话浓度检测

检测文本中的黑话浓度：

```
@jargon-pua 检测这段话的黑话浓度
@jargon-pua 分析一下这个文档的大厂风格程度
```

### 6. 循环优化模式

启动内卷循环，持续优化：

```
@jargon-pua 用循环模式优化这段代码
@jargon-pua 启动Loop，帮我优化这个功能，PUA强度适中
@jargon-pua 开启优化循环，不要太激进，用安全模式
```

## ⚙️ 强度控制

### PUA强度

| 级别 | 描述 | 使用场景 |
|------|------|----------|
| **mild** | 轻度建议 | 新手适应，温和反馈 |
| **moderate** | 中度批评 | 标准体验，正常反馈 |
| **strong** | 重度批评 | 进阶体验，严厉反馈 |
| **extreme** | 极度施压 | ⚠️ 需要确认，强烈情绪 |

### 黑话强度

| 级别 | 浓度 | 描述 |
|------|------|------|
| **light** | 5-15% | 点缀性质，偶尔使用 |
| **medium** | 15-30% | 明显风格，常用句式 |
| **heavy** | 30-50% | 大量使用，强烈风格 |
| **extreme** | 50%+ | ⚠️ 满篇黑话，人话难懂 |

### 使用示例

```
# 设置强度
@jargon-pua 设置PUA强度为轻度
@jargon-pua 黑话浓度30%
@jargon-pua 用温和一点的语气

# 组合控制
@jargon-pua 开启黑话模式，浓度中等，PUA强度轻
@jargon-pua 用循环模式优化，不要太激进
```

## 🛡️ 安全保护

### 过度优化防护

系统会自动检测：
- 单次改动行数过多
- 代码复杂度异常增加
- 核心功能被删除
- 可读性严重下降

### 使用建议

```
✅ 推荐用法：
- 新手从低强度开始
- 重要代码启用安全模式
- 设置合理的循环上限
- 及时保存工作版本

❌ 避免：
- 直接使用极高强度
- 在重要代码上无限制循环
- 忽略系统警告
```
